Альтернативный каталогизатор избранных публикаций

Решил опубликовать свое расширение, которое уже определенное время использую в качестве альтернативы.
В текущей реализации раздела избранного на сайте, недостаточно удобно реализован поиск данных (его нет)
Требовалась возможность просмотра давно внесенных данных, выгрузив все данные сразу, плюс хотелось иметь возможность задампить что-нибудь у любого пользователя
и оперативно пролистать \ скачать все изображения, а не по 10 элементов как сейчас.

Функционал

Сохранение публикаций и комментариев
Добавлен раздел на сайте с альтернативным каталогом сохраненных изображений (данные храняться в локальном хранилище браузера; формат хранения данных JSON)
Добавлена возможность импорта избранных публикаций из основного раздела сайта http://joyreactor.cc/user/[ник пользователя]/favorite

Управление профилями данных :

Можно хранить данные в нескольких хранилищах - профилях. Профили можно переключать \ сохранять \ удалять \ переносить в другой браузер \ пк. 
Актуально при импорте данных нескольких пользователей. Переключаться между ними при необходимости.
Импорт \ Экспорт данных в файл

Поддержка :

FireFox \ Chrome \ Opera - полная поддержка

Edge - Фунционал ограничен. Расшрирение функционирует, но API данного браузера сильно урезано. Функции сохранения данных через API заблокированы, а без этого не реализовать некоторые принципиально важные
моменты при скачивании изображений. 

Замечания :

При импорте данных время между итерациями скачивания страниц избранного от 2-30 (паузы) секунд. В общем случае время выгрузки равно 2-3 сек. * кол-во страниц. 
До публичного релиза этот функционал работал корректно в течении года. Если, что то вдруг перестанет функционировать, значит вероятно были внесены какие-то доп. ограничения. 
 
Не корректировать рекомендуемые параметры, любая подозрительная активность может быть расценена администрацией ресурса как спам атака с последующей временной \ постоянной блокировкой источника.

Расширения не использует какие-либо дополнительные сторонние библиотеки \ не лезет на левые ресурсы. Только расширение функцонала конкретного сайта.
